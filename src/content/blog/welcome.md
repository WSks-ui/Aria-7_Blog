新版博客暂时定名为 **Aria-7th Lab**。这个名字更适合放在首页主视觉里：英文短、识别度高，也能保留一点“个人实验室”的感觉。中文说明会更多放在 About、页脚和文章内容里，不抢第一屏。

这次没有继续沿用完整主题模板，而是用 Astro 重新搭一个静态站点骨架。页面交互尽量使用原生 JavaScript 和 CSS，保留轻量、可部署、好维护这几个前提。

## 首页先确定气质

首页的方向不是传统博客列表，而是一个偏漫画、二次元、个人剪贴簿的入口页。

当前首页已经完成了这些内容：

- 动态背景和遮罩，让主视觉更像一张柔和的桌面壁纸。
- 胶囊形个人标签墙，并加入轻量物理落下效果。
- 右侧滑动提示条，用来提示可以进入下一屏。
- 左侧 Aria Dock，承载音乐播放器和页面工具。
- 点击终端卡片后展开命令提示符窗口，头像会弹出并给出随机小纸条。
- 自定义光标、禁止右键菜单、长按复制和右键返回首页的小提示。

这些交互的目标不是炫技，而是让页面像一个可以被触碰的小空间。

## 为什么不用截图里的素材

参考图只用来确定风格方向，不直接复刻角色、动作、漫画分镜或文字。背景图和页面元素都尽量做成原创组合，避免把别人的视觉资产直接搬进站点。

后续如果继续换背景，也会优先选择：

- 自己生成或自己绘制的原创图。
- 授权明确的素材。
- 不会和具体角色动作高度相似的构图。

## 博客内容迁移

旧版 Firefly 博客里的文章已经迁移了一部分到新版内容集合里，包括 AI 绘图平台、Arch Linux 安装、MiniMind-O 本地推理、B 站音乐播放器和自定义光标等记录。

新版使用 Astro 内容集合管理文章，frontmatter 里保留标题、描述、发布日期、标签、分类和封面图。后面会继续补：

- 归档页。
- 标签页和分类页。
- 文章目录。
- 上一篇 / 下一篇。
- 相关文章。
- RSS 和站点地图。

这些是博客长期可用的基础设施，优先级会比继续堆首页装饰更高。

## Works 和 Playroom

Works 页面现在改成了 GitHub 仓库卡片，会直接跳转到对应项目。这样作品页不再只是占位文本，而是能展示真实项目线索。

Playroom 则先放了一个柔和风格的国际象棋小房间。规则使用 `chess.js` 处理，界面和交互仍然是原生 JS/CSS。它不追求复杂 AI，只作为博客里一个可以放松一下的小玩具。

## 接下来

短期内最应该继续补的是搜索、归档、标签页和文章页阅读体验。等这些基础能力稳定以后，再考虑相册、留言、评论或更多旧版功能。

这个站点现在还在成长期，但方向已经比较清楚：它应该是 Aria-7 的个人剪贴簿、项目记录和一点点二次元日常的入口。
